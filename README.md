# Лабораторная работа №1: Прямые методы решения СЛАУ и анализ их вычислительной устойчивости

Реализация, тестирование и сравнительный анализ прямых методов решения систем линейных алгебраических уравнений (СЛАУ) вида $Ax = b$ в соответствии с требованиями технического задания.

---

### Цели проекта
* Реализация набора функций для численного решения СЛАУ методами Гаусса и LU-разложения.
* Проведение сравнительного анализа времени работы алгоритмов.
* Экспериментальное демонстрация вычислительного преимущества LU-разложения при решении систем с одинаковой матрицей, но разными правыми частями.

### Задачи проекта
1. **Решение СЛАУ методом Гаусса без выбора ведущего элемента**.
2. **Решение СЛАУ методом Гаусса с выбором главного элемента по столбцу** (частичный выбор).
3. **LU-разложение матрицы** (выполняется без перестановок строк).
4. **Решение треугольных систем** (реализация процедур прямой и обратной подстановок для использования совместно с LU-разложением).
5. **Разработка исследовательской инфраструктуры**: написание высокоточного таймера, генераторов случайных матриц с фиксированным `seed` и утилит вычисления невязок.

---

## Математический базис методов

### 1. Метод Гаусса без выбора ведущего элемента (`solve_no_pivot`)
Классический алгоритм последовательного исключения переменных. На каждом шаге $k$ элементы строк $i > k$ обнуляются по формуле:

$$A_{ij} \leftarrow A_{ij} - \frac{A_{ik}}{A_{kk}} A_{kj}$$

$$b_i \leftarrow b_i - \frac{A_{ik}}{A_{kk}} b_k$$

* **Ограничение:** Метод вычислительно неустойчив. Если диагональный элемент $A_{kk}$ (ведущий элемент) равен или близок к численной границе нуля ($< 10^{-12}$), алгоритм прерывается с ошибкой `Zero pivot`.

### 2. Метод Гаусса с частичным выбором ведущего элемента (`solve_partial_pivot`)
Модификация для повышения численной устойчивости. Перед исключением переменных на шаге $k$ выполняется поиск максимального по модулю элемента в текущем столбце среди строк от $k$ до $n$:

```math \text{max\_row} = \arg\max_{i \ge k} |A_{ik}| ``` 

Если ```math \text{max\_row} \neq k ```, текущая строка $k$ и строка ```math \text{max\_row} ``` меняются местами в матрице $A$ и векторе $b$. 
* **Значение:** Гарантирует, что множитель исключения ```math |A_{ik}/A_{kk}| \le 1 ```, предотвращая экспоненциальный рост элементов и катастрофическое накопление погрешностей округления.


### 3. LU-разложение матрицы без перестановок (`LuSolver`)
Матрица коэффициентов $A$ раскладывается на произведение двух треугольных матриц: $A = L \cdot U$, где:
* $L$ — нижняя треугольная матрица с единицами на главной диагонали ($L_{ii} = 1$).
* $U$ — верхняя треугольная матрица (результат прямого хода исключения Гаусса).

Решение исходной системы $Ax = b$ декомпозируется на две последовательные треугольные системы со сложностью $O(N^2)$ каждая:
1. **Прямая подстановка (Forward Substitution):** Решение системы $Ly = b$ относительно промежуточного вектора $y$.
2. **Обратная подстановка (Backward Substitution):** Решение системы $Ux = y$ относительно искомого вектора $x$.

## Инструкция по сборке и запуску

Проект разработан на языке **C++** (стандарт **C++17**) с использованием кроссплатформенной системы автоматизации сборки **CMake**.

### Сборка через консоль (Linux / macOS / MinGW на Windows)

1. Находясь в корневом каталоге `Lab_1_LinAl/`, создайте директорию сборки и перейдите в нее:
   ```bash
   mkdir build && cd build
   ```
2. Сгенерируйте конфигурационные файлы сборки:
   ```bash
   cmake ..
   ```
3. Скомпилируйте исполняемое приложение (автоматически применятся флаги оптимизации кода `/O2` или `-O2` для корректных бенчмарков времени):
   ```bash
   cmake --build .
   ```
4. Запустите скомпилированное приложение:
   ```bash
   ./gauss_solver_app
   ```

### Сборка в IDE (Visual Studio / CLion / VS Code)
Вы можете открыть папку проекта `Lab_1_LinAl` как рабочий каталог CMake. Среда разработки автоматически подтянет файл `CMakeLists.txt`, выполнит индексацию проекта и позволит запустить исполняемый таргет `gauss_solver_app` через встроенную кнопку запуска (Run).

**Выводы по Эксперименту 4.2:**
Эксперимент полностью подтверждает ключевое преимущество LU-разложения на системах с инвариантной матрицей $A$. При $K=100$ достигнуто **почти 40-кратное ускорение вычислительного процесса**, что критически важно в задачах динамики, механики и моделирования.

1. Матрица Гильберта является классическим примером жестко плохо обусловленной системы, число обусловленности $\text{cond}(H)$ которой растет экспоненциально с ростом $N$. Это доказывает неприменимость стандартных прямых методов к плохо обусловленным задачам без предварительной регуляризации матрицы (например, регуляризации Тихонова) или перехода на типы данных повышенной точности.

## Заключение

В ходе выполнения лабораторной работы были успешно реализованы, протестированы и аналитически сопоставлены прямые методы численного решения систем линейных алгебраических уравнений (СЛАУ). На основе полученных экспериментальных данных сформулированы следующие ключевые выводы:

1. **Эмпирическое подтверждение сложности алгоритмов:** Эксперимент 4.1 наглядно продемонстрировал кубический рост временных затрат ($O(N^3)$) для всех исследуемых методов при увеличении размерности системы. Для одиночных СЛАУ общего вида методы Гаусса и LU-разложение обладают эквивалентной вычислительной сложностью, так как фаза построения верхнетреугольной матрицы $U$ алгоритмически идентична прямому ходу исключения переменных.
2. **Численная устойчивость и выбор ведущего элемента:** Сравнительный анализ базового метода Гаусса и модификации с частичным выбором по столбцу показал, что алгоритм `Partial Pivot` является необходимым стандартом при решении реальных задач. За счет исключения деления на малые диагональные элементы он предотвращает катастрофическое накопление погрешностей округления чисел с плавающей точкой (`double`), сохраняя строгий машинный ноль невязки там, где метод без выбора начинает терять точность значащих разрядов.
3. **Вычислительный триумф LU-подхода:** Эксперимент 4.2 полностью подтвердил теоретическую гипотезу о высокой эффективности LU-декомпозиции при работе со множественными правыми частями. Разделение тяжелой фазы исключения переменных ($O(N^3)$) и быстрой фазы подстановок ($O(N^2)$) позволило достичь **почти 40-кратного ускорения** процесса расчетов при $K=100$. Это делает LU-разложение фундаментальным базисом для моделирования динамических систем с инвариантной структурой связей.
4. **Проблема плохой обусловленности:** Стресс-тестирование на матрицах Гильберта (Эксперимент 4.3) наглядно проиллюстрировало явление вычислительной неустойчивости. Экспоненциальный рост числа обусловленности $\text{cond}(H)$ приводит к тому, что при $N \ge 10$ методы теряют точность, а при $N=15$ полностью разрушаются, приводя к делению на числа за границей точности типа `double`. Полученные результаты доказывают, что малая норма невязки является лишь необходимым, но недостаточно обоснованным критерием корректности численного ответа, а плохо обусловленные системы требуют применения специализированных методов регуляризации или перехода к итерационным подходам.

Разработанный программный комплекс полностью удовлетворяет критериям модульности, кроссплатформенности и численной воспроизводимости результатов, заложенным в техническом задании.
